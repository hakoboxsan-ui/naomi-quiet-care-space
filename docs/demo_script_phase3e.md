# NAOMI Phase 3-E デモスクリプト (60〜90秒体験用)

このスクリプトは、NAOMIのコアバリューである「状態整理」「低圧応対」「人へのパス」を短時間で確実に伝えるためのものです。

## 1. 導入 (10秒)
「NAOMIは、普通のAIのように『すぐ解決策を出す』のではなく、『相手の状態を見て接し方を変え、情報を整理する』AIです。百聞は一見に如かず、実際に見ていただきましょう」

## 2. 普通のAIとの違い ＆ 会話圧の低下 (20秒)
**【操作】Tab 1の「限界+助言」ボタンをクリック**
「例えば、『どうしたらいいか教えてほしいけど、正直もう疲れてる』という言葉。普通のAIは下のように、解決策やタスクを一方的に提示してしまいます。これは疲れている人には逆効果です。」
「一方、NAOMIはユーザーの状態（疲労感や助言要求）を瞬時に推定し、『会話圧を低くする』戦略をとります。『すぐに答えを押しつけるより、まず状況を一緒にほどいていこうか』と、まずは受け止める余白を作ります」

## 3. より深い低圧応対と普段との違い (15秒)
**【操作】Tab 2の「思考過多：眠れない夜」ボタンをクリック**
「さらに別のシナリオを見てみます。NAOMIはユーザーの普段の状態（ベースライン）を知っているため、『いつもより少し張りつめている様子があります』といった自然な気遣いが可能です。解決を急がず、認知的休息（Cognitive Relief）を提供します」

## 4. 状態整理と Staff Note (25秒)
**【操作】Tab 2の「状態整理モード（内科領域）を開始」をクリックし、1問回答して終了**
「そして、ここからが現場で使える機能です。NAOMIはただ優しく雑談するだけではなく、状態を整理するための『Intake（問診）モード』にシームレスに移行できます」
**【操作】Staff Note（状態整理メモ）エリアを指す**
「会話の結果は『Staff Note』としてまとめられます。これはAIによる『医療診断』ではなく、あくまで『客観的な状態整理』です。」
「これにより、介護施設、病院の受付、企業の相談窓口などで、『本人の負担を最小限に抑えつつ、スタッフが状況を把握できる形でパスする』という前段の補助AIとして機能します」

## 5. 結び (5秒)
「NAOMIは、人を診断して正論で打ち負かすAIではなく、人の状態に寄り添い、現場の支援者につなぐための『低圧な整理補助AI』です」
