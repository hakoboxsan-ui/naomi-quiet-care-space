# NAOMI 音声設計メモ (Voice Accessibility Plan)

本ドキュメントは、高齢者や視覚障害者、および文字入力や読み取りに負担を感じるユーザーのための音声案内設計方針についてまとめたものです。

## 1. 音声案内の基本役割
- **視覚補助と道標（サインポスト）**
  高齢者や視力の弱いユーザーにとって、画面上の小さなテキストやボタンを探し出すことは大きな認知的負荷になります。音声は、現在のシステム状態や次に選択可能なアクション（例：「お辛い時は、画面下部の『つらい』ボタンを押してくださいね」など）を優しく案内する道標として機能します。
- **認知的休息（Cognitive Relief）の促進**
  テキストの読解が難しい状況においても、温かみのある声で語りかけることにより、緊張を和らげ、安心感を与えます。

## 2. 音声合成（TTS）モデル候補
NAOMIの静寂で奥ゆかしい世界観（Quiet Luxury）を表現するため、以下のモデルを将来の統合候補とします。
- **Noa (Style-BERT-VITS2)**
  非常に自然で感情豊かな日本語合成が可能なStyle-BERT-VITS2向けの音声モデル（Noa.pth / Noa.index）。NAOMIの持つ「静かで、急かさない、そっと寄り添う」パーソナリティを最も高精度に表現できます。
- **irodori_TTS**
  軽量でありながら感情表現に優れ、リアルタイム対話システムへの統合が容易な日本語音声合成エンジン。

## 3. 音声デザイン原則（低圧・非干渉）
NAOMIの音声は、一般的な情報伝達型AI（スマートスピーカーなど）の早口で冷たいトーンとは明確に一線を画します。

- **早口の厳禁**:
  発話速度を意図的に落とし、高齢者でも十分に聞き取れるよう設計します。
- **短文・簡潔さ**:
  一度に大量の情報を話さず、一回あたりの発話は1〜2文の短い構成にします。
- **低圧（Low Pressure）**:
  高音でハキハキとした声ではなく、少し低めで穏やかなトーン（ウィスパーボイスに近い優しさ）を採用します。
- **「急かさない」対話設計**:
  ユーザーに返答を無理強いするような問いかけを避け、「いつでも、話したくなったときにボタンや声で教えてくださいね」という余白を伝える発話構成を徹底します。
