# NAOMI カメラ視覚情報連携・将来設計メモ (Future Vision Integration Plan)

本ドキュメントは、タッチや音声といった入力手段に加え、視覚情報（カメラ入力）を通じて利用者の状態を適応的に察知するための将来設計についてまとめたものです。MVP（最小限の動作デモ）の段階ではカメラ機能の実装は行わず、設計構想のみにとどめます。

## 1. コア思想：「見張るAI」ではなく「気づくAI」
医療現場や介護施設において、カメラを使用した監視システムは「常時監視されている」という精神的ストレス（会話圧の上昇）をユーザーに与えがちです。
NAOMIのカメラ設計においては、監視やデータ管理を主目的とせず、あくまで「会話のきっかけをそっと得るための気づき」としてカメラ情報を扱います。

- **常時監視の禁止**
  利用者の顔や行動をずっと監視・録画するシステムにはしません。
- **プライバシー保護第一**
  画像・映像データはサーバーに送信せず、ローカル（エッジ側）で瞬時に特徴量（姿勢・表情の傾向）へ変換し、元の映像データは即時破棄する構成とします。

## 2. センシング対象と非断定の原則
カメラから取得する情報は、ユーザーの状態を決めつけるためではなく、状態整理をサポートする「補助的なヒント」としてのみ使用します。

- **表情（FACS等の活用）**
  眉の寄せ方や口元の緊張などを表情分析（Facial Action Coding System）の観点で簡易推定しますが、「あなたは怒っています」「悲しんでいます」と決めつける応答は行いません。
  「少し肩や表情がこわばっている様子が見受けられますが、お疲れではありませんか？」などの極めて低圧なアプローチのフックとします。
- **視線と注意力**
  画面を注視しているか、視線が泳いでいるか等の変化から、疲労や認知的な負荷を推定します。
- **姿勢と動作**
  前かがみになっている、顔を手で覆っているといった姿勢から、エネルギー低下や頭痛・身体的苦痛の可能性を優しく察知します。

## 3. 実装のロードマップ（段階的拡張）
- **Phase 4-A (現在)**:
  モックアップUIでのアクセシビリティボタン入力と状態シミュレーションのみ。カメラ実装は行いません。
- **Phase 4-B (将来)**:
  ローカルPC上のWebカメラを使用した、ローカルFACS / 表情特徴量の抽出実験（完全にクローズドな検証環境）。
- **Phase 4-C (将来)**:
  プライバシー承諾を得た特定の検証デバイス（例: 介護サポート端末）での実証運用テスト。
